# Fresco New Tab

A Manifest V3 browser extension that replaces the new-tab page with your
current [Fresco](https://github.com/) wallpaper — a full-viewport still of
whatever is on your desktop (or a browser-specific pick), a large clock, and
nothing else. Plain HTML/CSS/JS, no build step, no frameworks.

## Requirements

- Fresco must be running on this machine, with the **Browser new-tab** setting
  enabled in Fresco's Settings. That starts a local bridge on
  `http://127.0.0.1:8765` which serves the current wallpaper frame.
- If the bridge is unreachable the page falls back to a dark gradient with a
  working clock and a small dismissible hint.

## Load unpacked

**Chrome / Edge**

1. Open `chrome://extensions` (Edge: `edge://extensions`).
2. Enable **Developer mode** (top right).
3. Click **Load unpacked** and select this `extension/` directory.

**Brave**

1. Open `brave://extensions`.
2. Enable **Developer mode** and click **Load unpacked**, selecting this
   directory. Standard MV3 — nothing Brave-specific is required.
3. Shields note: requests go only to `127.0.0.1`, which Brave Shields does not
   block, so the extension works with Shields at default settings.

**Firefox**

1. Open `about:debugging#/runtime/this-firefox`.
2. Click **Load Temporary Add-on…** and pick `manifest.json` in this
   directory.

The manifest already contains a `browser_specific_settings.gecko` block with
the id `newtab@fresco.app`, which Firefox requires and Chromium browsers
ignore (Chrome may log a harmless "unrecognized key" warning). One behavioral
difference: Firefox uses `chrome_url_overrides.newtab` too, so the same
manifest serves all four browsers — no fork needed. If you package for AMO,
note that Firefox's MV3 treats `host_permissions` as optional-by-default; on
first use you may need to grant the `127.0.0.1` permission from the
extension's entry in `about:addons`.

## Privacy

- Talks only to `http://127.0.0.1:8765` (the local Fresco bridge).
- Zero external network requests. Zero tracking, analytics, or storage beyond
  one localStorage flag remembering that you dismissed the hint.

## Icons

`icons/icon{16,48,128}.png` were generated by a small stdlib-only Python
script (rounded #0284c7 square, white "f", 4x supersampled). Regenerate at any
size by re-running the same approach, or recreate from a vector editor —
nothing depends on the exact pixels.
